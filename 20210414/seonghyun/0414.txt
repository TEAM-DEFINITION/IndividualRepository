문제점
	1. 패킷 유실 (네트워크 이슈 등)
	2. 사용자 인증 문제
	3. 반복적 공격
	4. genesis 블록 생성 문제
	5. 클라이언트 기기의 로컬 데이터 암호화 필요성
	6. 데이터 송, 수신시의 데이터 암, 복호화 문제

방안
	1. 본인 인증 api
	2. 체인 검증
		- 체인 리셋
		- 유효한 체인 재사용
	3. 요청에 관한 시간, 횟수로 판단
		- 재전송 요청
		- ID 차단
		- 체인 검증
	4. 로컬 데이터 암호화 방식 추가
	5. 플랫폼 간의 암, 복호화 방식 및 포맷 확인, 수정
	6. 인증 과정 비대칭키 생성 -> 안드로이드 키스토어 api 활용

프로젝트 관련
	1. 각 기능 및 ui에 대한 역할 분담 필요성
	2. 플랫폼 선정
